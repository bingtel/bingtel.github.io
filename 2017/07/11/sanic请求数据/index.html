请求数据

当一个端点收到一个HTTP请求，路由器函数就会被传入一个Request对象。

下面的变量作为Request对象的属性，是可以访问的：

  • json(任何) - JSON体

      from sanic.response import json

  @app.route("/json")
  def post_json(request):
      return json({ "received": True, "message": request.json })

  • args(字典 - 查询参数中的变量。查询字符串是URL中形式如?key1=value1&key2=value2的部分。如果URL被解析了，那么args字典看起来就是{'key1': ['value1'], 'key2': ['value2']}。request的query_string参数包含未被解析的字符串值。

      from sanic.response import json

  @app.route("/query_string")
  def query_string(request):
      return json({ "parsed": True, "args": request.args, "url": request.url, "query_string": request.query_string })

  • raw_args(字典) - 在许多情况下，您将需要访问一个较少打包的字典中的url参数。还以前面的URL?key1=value1&key2=value2说事，raw_args字典看起来就如同： {'key1': 'value1', 'key2': 'value2'}

  • files(File对象的字典) - 具有名称，正文和类型的文件列表。

      from sanic.response import json

  @app.route("/files")
  def post_json(request):
      test_file = request.files.get('test')

      file_parameters = {
      'body': test_file.body,
      'name': test_file.name,
      'type': test_file.type,
      }

      return json({ "received": True, "file_names": request.files.keys(), "test_file_parameters": file_parameters })

  • form(字典) - 被传递的表单变量

      from sanic.response import json

  @app.route("/form")
  def post_json(request):
      return json({ "received": True, "form_data": request.form, "test": request.form.get('test') })

  • body(字节流) - 原始的请求体。此属性允许检索请求的原始数据，不管content-type如何。

      from sanic.response import text

  @app.route("/users", methods=["POST",])
  def create_user(request):
      return text("You are trying to create a user with the following POST: %s" % request.body)

  • ip(字符串) - 请求方的IP地址

  • app - 对正在处理此请求的Sanic应用程序对象的引用。当在蓝图中或者其它无法访问全局应用对象的处理程序中时，这是非常有用的。

      from sanic.response import json
  from sanic import Blueprint

  bp = Blueprint('my_blueprint')

  @bp.route('/')
  async def bp_root(request):
      if request.app.config['DEBUG']:
      return json({'status': 'debug'})
      else:
      return json({'status': 'production'})

  • url: 请求的完整URL，如： http://localhost:8000/posts/1/?foo=bar

  • scheme: 请求的协议， http, https

  • host: 请求的host地址， 如： localhost:8080

  • path： 请求的路径， /posts/1/

  • query_string: 请求的查询参数， foo=bar 或者''

使用get和get_list获取参数

返回字典的请求属性实际上返回称为RequestParameters的dict的子类。使用此对象的主要区别是get和getlist方法之间的区别。

  • get(key，default=None)正常运行，除了当给定键的值是列表时，只返回第一个项目。

  • getlist(key，default=None)正常运行，返回整个列表。
