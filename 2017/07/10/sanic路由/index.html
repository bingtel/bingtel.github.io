请求参数

Sanic配备了支持请求参数的基本路由器。

为了指定一个参数，可以用”<>”包裹它，如<parameter>.请求参数会被作为关键字参数传给路由处理器。

为了指定参数的类型，在”<>”里面，在参数名称的后面加上”:type”即可。如果参数和指定的类型不匹配，Sanic会抛出NotFound的异常，导致在指定的URL页面中显示404: Page not found的错误。

HTTP请求的类型

默认情况下，一个URL上定义的路由，只接受以GET方式访问该URL的请求。然而，@app.route装饰器，接受一个可选的methods参数，该参数会允许处理器函数处理列表(methods对应的值是一个list)指定的所有的HTTP方法。

还有一个可选的host参数(既可以是一个list也可以是string)。这会把一个路由限制到指定的主机上。如果有一个没有指定host的路由，该路由会是默认的路由。

使用url_for构建URL

Sanic提供了url_for方法，能够基于处理器的方法名称来构建对应的URL。这对于避免在应用中硬编码URL很有用；取而代之的是，你可以通过方法名称来引用URL。例如：

使用url_for时，还要记住：

  • 传给url_for的关键字参数如果不是请求参数，将会出现在URL的查询参数中，如：

      url = app.url_for('post_handler', post_id=5, arg_one='one', arg_two='two')
  # /posts/5?arg_one=one&arg_two=two

  • 多值参数也可以传给url_for, 如：

      url = app.url_for('post_handler', post_id=5, arg_one=['one', 'two'])
  # /posts/5?arg_one=one&arg_one=two

  • 一些特殊的参数(_anchor, _external, _scheme, _method, _server)传给url_for时，会有特殊的URL构建语义(_method当前不被支持，会被忽略)。如：

      url = app.url_for('post_handler', post_id=5, arg_one='one', _anchor='anchor')
  # /posts/5?arg_one=one#anchor
  
  url = app.url_for('post_handler', post_id=5, arg_one='one', _external=True)
  # //server/posts/5?arg_one=one
  # _external requires passed argument _server or SERVER_NAME in app.config or url will be same as no _external
  
  url = app.url_for('post_handler', post_id=5, arg_one='one', _scheme='http', _external=True)
  # http://server/posts/5?arg_one=one
  # when specifying _scheme, _external must be True
  
  # you can pass all special arguments one time
  url = app.url_for('post_handler', post_id=5, arg_one=['one', 'two'], arg_two=2, _anchor='anchor', _scheme='http', _external=True, _server='another_server:8888')
  # http://another_server:8888/posts/5?arg_one=one&arg_one=two&arg_two=2#anchor

  • 所有有效的参数都要传给url_for来构建URL。如果有参数未提供、或者参数和指定的类型不匹配，URLBuildError错误会被抛出。

WebSocket路由的处理器作为第一个参数传递请求，WebSocket协议对象作为第二个参数。协议对象分别具有send和recv数据的发送和接收方法。
