判断对象不为None的坑

发表于 2017-08-14 | 分类于 python

python开发中，经常要判断对象是否非空，写：

if obj is not None:

肯定是没问题的，但是有时候，会写成：

if obj:

，这就可能会带来错误，一些看起来不是None的对象，会让

if obj:

返回False。

那是因为：

if obj:

等同于:

if bool(obj):

如果该对象的类中，重新定义了:

__bool__()方法

那么，if bool(obj):就有可能返回False, 即便此时obj is not None.

比如， 近日在使用grequests库(gevent + requests)来发起请求时，返回了一个403的Response对象，我用了：

if resp:

来判断响应是否为空，结果老是False, 无法执行该if分支，看了代码(requests.models.Response类)才发现：

__bool_方法已经被重写了，200-400之间的响应才是True，否则就是False;

使用别人的代码，一定要谨慎～
