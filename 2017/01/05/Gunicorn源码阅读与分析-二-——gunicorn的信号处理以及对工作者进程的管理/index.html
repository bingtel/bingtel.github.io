Gunicorn源码阅读与分析(二)——gunicorn的信号处理以及对工作者进程的管理

发表于 2017-01-05 | 分类于 gunicorn

在上文Gunicorn源码阅读与分析-一-——Gunicorn的整体运行轨迹中提到了gunicorn的整体运行轨迹，知道了:

主进程接收信号并管理工作者进程；工作者进程负责接收和处理请求。

那下面就看一下, 主进程是如何处理接收信号并对工作者进行管理的。

主进程的初始化

这就涉及到两部分：

1. 主进程对象的实例化： 主要完成了初始化主进程对象的准备工作
2. 启动主进程对象： 主要是初始化工作者进程

下面就从这两个方面介绍一下。

这里可以看到，主进程的一些变量的声明和初始化，如：worker_age、START_CTX等。
也找到了我们要继续跟踪的方法：self.setup(app)。

在setup()方法中，先通过

self.app = app

让主进程获得了BaseApplication应用对象(用于工作者进程和应用通信)，并通过：

self.cfg = app.cfg

获得了gunicorn启动时的配置参数信息，然后利用这些cfg信息，初始化一些属性：

self.worker_class = self.cfg.worker_class
self.address = self.cfg.address
self.num_workers = self.cfg.workers
self.timeout = self.cfg.timeout
self.proc_name = self.cfg.proc_name

并设置环境变量：

if self.cfg.env:
for k, v in self.cfg.env.items():
    os.environ[k] = v

最后，判断是否要预加载应用：

if self.cfg.preload_app:
    self.app.wsgi()

到这里， 主进程对象本身的初始化的工作就结束了， 下面介绍一下工作者进程的初始化问题。

可以看出，run()方法的流程比较清晰：

1. 调用start()初始化主进程，必要的时候监听和设置主进程pidfile
