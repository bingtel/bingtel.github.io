自动创建flask项目

发表于 2016-12-03 | 分类于 flask

自己跟着《Flask Web开发》一书自学Flask框架的使用，觉得flask基于python和众多的插件，开发速度确实远过于JavaEE中的Spring MVC和JFinal框架。但是Spring MVC中的Controller和@RequestMapping却有助于自己理解flask中的BluePrint概念。

自己觉得《Flask Web开发》中的第七章中，对于项目的组织结构的示例说明很是清晰，值得参考，下面用Shell脚本来实现项目组织结构的自动创建，以后可以加快新建项目的开发工作和约定。

init_app.sh脚本

使用命令

./init_app.sh  ProjectName


#!/bin/bash
if [ $# -ne 1 ]; then
    echo "错误: 只需要一个路径参数"
    exit 1
fi
echo "输入的路径是: $1"

mkdir $1
if [ $? -ne 0 ]; then
    echo "错误: $1 路径参数有问题"
    exit 1
fi

cd $1

mkdir app
mkdir app/templates
mkdir app/static

mkdir app/main
touch app/main/__init__.py
touch app/main/errors.py
touch app/main/forms.py
touch app/main/views.py

touch app/__init__.py
touch app/email.py
touch app/models.py

mkdir migrations
mkdir tests
touch tests/__init__.py
touch requirements.txt
touch config.py
touch manage.py
chmod 755 manage.py

touch sql.sql

virtualenv venv

source venv/bin/activate
pip install MySQL-python
pip install flask
pip install flask-script
pip install flask-migrate
pip install flask-wtf
pip install flask-mail
pip install flask-moment
pip install flask-bootstrap
pip install flask-sqlalchemy
pip install flask-login

项目目录说明 ProjectName/

    |----app/  

         |----email.py #邮件插件

         |----__init__.py  #包初始化

         |----models.py  #数据库Model

         |----static/  #css等静态文件

         |----templates/  #页面文件

         |----main/   #一个蓝图(蓝图的名称最好和文件夹名称相同；如果有多个蓝图,相应的文件夹和main保持同级)

              |----__init__.py  #包初始化

              |----errors.py  #蓝图的错误配置

              |----forms.py   #蓝图的表单

              |----views.py   #蓝图的视图

     |----manage.py          #启动项目

     |----config.py          #环境的常量配置

     |----migrations/        #数据迁移

     |----requirements.txt   #项目以来的插件 pip freeze > requirements.txt

     |----sql.sql            #sql

     |----tests/             #测试文件夹

     |----venv               #虚拟环境   通过source venv/bin/activate 启动
# 创建flask项目 # flask web
