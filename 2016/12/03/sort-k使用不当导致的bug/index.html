sort -k使用不当导致的bug

发表于 2016-12-03 | 分类于 linux

问题

项目开发过程中， 需要解决如下问题：

根据日志文件的第二列(时间戳)进行时间戳倒序排列， 并找出符合条件的行。

“找出符合条件的行”只需要使用awk过滤即可， 问题就出现在了：

根据日志文件的第二列(时间戳)进行时间戳倒叙排列。

我使用

tail -n 10000 log | sort -t, -nk  2 -r > t

来实现排序要求， 却发现并没有准确地按照时间戳来倒叙排列。

如：

那么问题出现在哪儿呢？

思考

容易想到， 要么：

sort算法有问题？

要么

排序的条件有问题

sort算法已经经过很多人的实践使用， 不会出现问题， 那么就只可能在：

排序的条件有问题

那么，

-n -r -t -k

4个参数中哪些有问题呢？

-n 按照数值比较
-r  倒序排列
-t  指定列的分隔符
-k  指定排序的列(这个说法很具有迷惑性，网上很多文章也这么说，随后说明)

解决

还好首先考虑了

-k

参数， 发现了问题，**-k**选项的语法格式，如下：

[ FStart [ .CStart ] ] [ Modifier ] [ , [ FEnd [ .CEnd ] ][ Modifier ] ]

这个语法格式可以被其中的逗号（，）分为两大部分，Start部分和End部分。

“如果不设定End部分，那么就认为End被设定为行尾”。

问题就出现在这， 在使用：

tail -n 10000 log | sort -t, -nk  2 -r > t

没有指定End, 导致排序结果不正确， 需要改为：

tail -n 10000 log | sort -t, -nk  2,2 -r > t
