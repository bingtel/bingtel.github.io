gunicorn官方文档翻译-安装

发表于 2016-12-03 | 分类于 gunicorn

Installation(安装)

Requirements: Python 2.x >= 2.6 or Python 3.x >= 3.2

要求： Python 2.x >= 2.6 或者 Python3.x >= 3.x**

To install the latest released version of Gunicorn:

为了安装Gunicorn最新的发布版本，

$ pip install gunicorn

From Source(从源码安装)

You can install Gunicorn from source just as you would install any other Python package:

你可以像安装其它的Python包一样从源码安装Gunciron，

$ pip install git+https://github.com/benoitc/gunicorn.git

This will allow you to keep up to date with development on GitHub:

这会让你和GitHub上的开发保持一致。

$ pip install -U git+https://github.com/benoitc/gunicorn.git

Async Workers(异步工作者)

You may also want to install Eventlet or Gevent if you expect that your application code may need to pause for extended periods of time during request processing. Check out the design docs for more information on when you’ll want to consider one of the alternate worker types.

当你打算让你的应用在处理请求的时候能够暂停一段时间，你也许想安装Eventlet或者Gevent。当你考虑工作者类型备选项中的一种时， 检查一下相应的设计文档来获取更多的信息。

$ pip install greenlet  # Required for both
$ pip install eventlet  # For eventlet workers
$ pip install gevent    # For gevent workers

Note(注意)

If installing greenlet fails you probably need to install the Python headers. These headers are available in most package managers. 
On Ubuntu the package name for apt-get is python-dev.

如果安装greenlet失败， 你很大程度上需要安装Python headers。 这些headers能够从绝大多数的包管理器中获得。 在Ubuntu系统中， 对于apt-get而言， 该header包的名字是**python-dev**.

Gevent also requires that libevent 1.4.x or 2.0.4 is installed. This could be a more recent version than what is available in your package manager. 
If Gevent fails to build even with libevent installed, this is the most likely reason.

Gevent也要求libevent 1.4.x 或者 2.0.4的版本提前安装。 这些版本在包管理器中是比较新的版本。 如果libevent安装了， Gevent还是编辑失败， 那么libevent的版本问题很可能就是问题所在。

Debian GNU/Linux

If you are using Debian GNU/Linux and it is recommended that you use system packages to install Gunicorn except maybe when you want to use different versions of Gunicorn with virtualenv. This has a number of advantages:

如果你在生使用 Debian GNU/Linux 系统， 推荐你使用系统包来安装Gunicorn， 除非你想在virtualenv中使用不同版本的Gunicorn。 此举有下列好处：

  • Zero-effort installation: Automatically starts multiple Gunicorn instances based on configurations defined in /etc/gunicorn.d.(易安装： 基于/etc/gunicorn.d中的配置， 自动启动多个Gunicorn实例。)
  • Sensible default locations for logs (/var/log/gunicorn). Logs can be automatically rotated and compressed using logrotate.(存放日志的默认位置(/var/log/gunicorn)是明智的： 通过logratate, 日志文件能够自动轮流和压缩。)
  • Improved security: Can easily run each Gunicorn instance with a dedicated UNIX user/group.(安全性已改善：能通过一个专用的用户/用户组来运行一个Gunicorn实例。)
  • Sensible upgrade path: Upgrades to newer versions result in less downtime, handle conflicting changes in configuration options, and can be quickly rolled back in case of incompatibility. The package can also be purged entirely from the system in seconds. stable (“jessie”) (清晰的升级路径：更新到较新的版本， 带来：更少的停用时间、处理配置项中的冲突变化、防止不兼容而快速回滚等好处。该包也可以被完全卸载)

The version of Gunicorn in the Debian “stable” distribution is 19.0 (June 2014). You can install it using:

Gunicorn在Debian”stable”发行版中的版本是19.0(截止到2014.6)。 可以使用下面的命令安装：

$ sudo apt-get install gunicorn

You can also use the most recent version by using Debian Backports. First, copy the following line to your /etc/apt/sources.list:

你也可以通过Debian Backports来使用最新的版本。第一步， 把下面一行拷贝到你的 /etc/apt/sources.list中：

 deb http://backports.debian.org/debian-backports jessie-backports main

Then, update your local package lists:

接着更新本地的包列表，

$ sudo apt-get update

You can then install the latest version using:

你可以使用下述命令安装最新的版本：

$ sudo apt-get -t jessie-backports install gunicorn
