Settings(设置)

This is an exhaustive list of settings for Gunicorn. Some settings are only able to be set from a configuration file. The setting name is what should be used in the configuration file. The command line arguments are listed as well for reference on setting at the command line.

这是一份费神的Gunicorn的设置列表。有些设置只能从配置文件中设置。设置项名称指的是应该在配置文件中使用的东西。为了以供参考，命令行参数也列出了。

Config File(配置文件)

config

  • -c CONFIG, --config CONFIG
  • None

The Gunicorn config file.

Gunicorn的配置文件。

A string of the form PATH, file:PATH, or python:MODULE_NAME.

PATH、file:PATH、python:MODULE_NAME的字符串形式。

Only has an effect when specified on the command line or as part of an application specific configuration.

只有当命令行中指定或者应用的配置中指定时，才会起作用。

Changed in version 19.4: Loading the config from a Python module requires the python: prefix.

19.4中，做出了改变：从Python模块中加载配置时，要求python:作为前缀

Server Socket

bind

  • -b ADDRESS, --bind ADDRESS

  • ['127.0.0.1:8000']

The socket to bind.

需要绑定的套接字。

A string of the form: HOST, HOST:PORT, unix:PATH. An IP is a valid HOST.

HOST、HOST:PORT、unix:PATH的字符串形式。IP是有效的HOST。

Multiple addresses can be bound. ex.:

可以绑定多个地址， 例如：

$ gunicorn -b 127.0.0.1:8000 -b [::1]:8000 test:app

will bind the test:app application on localhost both on ipv6 and ipv4 interfaces.

会把绑定test:app绑定到localhost的ipv4和ipv6的网口。

backlog

  • --backlog INT

  • 2048

The maximum number of pending connections.

最大连接数。

This refers to the number of clients that can be waiting to be served. Exceeding this number results in the client getting an error when attempting to connect. It should only affect servers under significant load.

该参数表示等待被服务的客户端的上限。超过此数目时， 尝试连接服务器的客户端会得到一个错误提示。在负载较高时， 该参数才会影响服务器。

Must be a positive integer. Generally set in the 64-2048 range.

必须是正整数， 通常介于64-2048之间。

Worker Processes

workers

  • -w INT, --workers INT

  • 1

The number of worker processes for handling requests.

用于处理请求的工作者进程数量。

A positive integer generally in the 2-4 x $(NUM_CORES) range. You’ll want to vary this a bit to find the best for your particular application’s work load.

一个通常介于CPU核数的2-4倍的正整数。你可以通过微调次参数来获取最适合你应用的值。

By default, the value of the WEB_CONCURRENCY environment variable. If it is not defined, the default is 1.

缺省情况下， 由WEB_CONCURRENCY环境变量指定。否则， 默认值是1.

worker_class

  • -k STRING, --worker-class STRING

  • sync

The type of workers to use.

要使用的工作者的类型。

The default class (sync) should handle most “normal” types of workloads. You’ll want to read Design for information on when you might want to choose one of the other worker classes.

默认的类(sync)能够处理大多数的请求负载。如果你想选择其他的工作类， 你应该读一下Design来获取更多的信息

A string referring to one of the following bundled classes:

工作类的集合如下：

  • sync
  • eventlet - Requires eventlet >= 0.9.7
  • gevent - Requires gevent >= 0.13
  • tornado - Requires tornado >= 0.2
  • gthread - Python 2 requires the futures package to be installed
  • gaiohttp - Requires Python 3.4 and aiohttp >= 0.21.5

Optionally, you can provide your own worker by giving Gunicorn a Python path to a subclass of gunicorn.workers.base.Worker. This alternative syntax will load the gevent class: gunicorn.workers.ggevent.GeventWorker. Alternatively, the syntax can also load the gevent class with egg:gunicorn#gevent.

作为可选项，你可以通过告知Gunicorngunicorn.workers.base.Worker的一个子类的路径来使用你自己的worker。这个替换的语法会加载gevent类：gunicorn.workers.ggevent.GeventWorker。也可以通过加载gevent类egg:gunicorn#gevent作为替代。

threads

  • --threads INT

  • 1

The number of worker threads for handling requests.

处理请求的工作线程数。

Run each worker with the specified number of threads.

为每个工作者进程指定的线程数。

A positive integer generally in the 2-4 x $(NUM_CORES) range. You’ll want to vary this a bit to find the best for your particular application’s work load.

一个介于CPU核数的2-4倍的正整数。可以通过微调该参数，来获取最适合自身应用的值。

If it is not defined, the default is 1.

默认是1.

worker_connections

  • --worker-connections INT

  • 1000

The maximum number of simultaneous clients.

同时连接的客户端数目上限。

This setting only affects the Eventlet and Gevent worker types.

该配置只会影响Eventlet、Gevent工作者类型。

max_requests

  • --max-requests INT
  • 0

The maximum number of requests a worker will process before restarting.

在重启前， 一个工作者进程能够处理的最大请求数。

Any value greater than zero will limit the number of requests a work will process before automatically restarting. This is a simple method to help limit the damage of memory leaks.

一个大于0的值，将会限制一个工作者进程在重启前能够处理的请求数。这是一种减少内存泄漏的简单方法。

If this is set to zero (the default) then the automatic worker restarts are disabled.

如果该值是0(默认值)， 那么工作者进程不会默认重启。

max_requests_jitter

  • --max-requests-jitter INT

  • 0

The maximum jitter to add to the max_requests setting.

The jitter causes the restart per worker to be randomized by randint(0, max_requests_jitter). This is intended to stagger worker restarts to avoid all workers restarting at the same time.

New in version 19.2.

timeout

  • -t INT, --timeout INT

  • 30

Workers silent for more than this many seconds are killed and restarted.

工作者进程静默时间超过该时间， 会被杀死并重启。

Generally set to thirty seconds. Only set this noticeably higher if you’re sure of the repercussions for sync workers. For the non sync workers it just means that the worker process is still communicating and is not tied to the length of time required to handle a single request.

通常该值设为30秒。只有当你明白为sync工作者进程设置该值的后果时，才能设置该值。对于非sync工作者进程， 该参数仅意味着进程还在通信，而非限制处理单个请求的时间。

graceful_timeout

  • --graceful-timeout INT

  • 30

Timeout for graceful workers restart.

工作者重启的超时时间。

After receiving a restart signal, workers have this much time to finish serving requests. Workers still alive after the timeout (starting from the receipt of the restart signal) are force killed.

收到该信号后， 工作者还剩余这么多的时间来处理请求。 超过该时间后， 工作者进程会被强制杀死。

keepalive

  • --keep-alive INT

  • 2

The number of seconds to wait for requests on a Keep-Alive connection.

在一个设置了Keep-Alive的连接上， 等待一个请求的秒数。

Generally set in the 1-5 seconds range.

通常介于1-5秒之间。

Security

limit_request_line

  • --limit-request-line INT

  • 4094

The maximum size of HTTP request line in bytes.

HTTP请求的最大字节数。

This parameter is used to limit the allowed size of a client’s HTTP request-line. Since the request-line consists of the HTTP method, URI, and protocol version, this directive places a restriction on the length of a request-URI allowed for a request on the server. A server needs this value to be large enough to hold any of its resource names, including any information that might be passed in the query part of a GET request. Value is a number from 0 (unlimited) to 8190.

该参数用于限制客户端的HTTP请求行数。由于请求包含HTPP方法、URI、协议版本等，这个指令对服务器上的request-URI的长度做出了限制。服务器需要该值足够大，这样它能够容纳所有的资源名， 包括GET请求的所有能够解析的部分， 该值介于0-8199， 0表示不限制大小.

This parameter can be used to prevent any DDOS attack.

该值可用于阻止DDOS攻击。

limit_request_fields

  • --limit-request-fields INT

  • 100

Limit the number of HTTP headers fields in a request.

限制一个请求头的字段的个数。

This parameter is used to limit the number of headers in a request to prevent DDOS attack. Used with the limit_request_field_size it allows more safety. By default this value is 100 and can’t be larger than 32768.

该参数通过限制headers的个数来阻止DDOS攻击。和limit_request_field_size搭配使用， 更安全。 默认情况下， 该值是100， 单不能超过32768。

limit_request_field_size

  • --limit-request-field_size INT

  • 8190

Limit the allowed size of an HTTP request header field.

限制HTTP请求头域的大小。

Value is a positive number or 0. Setting it to 0 will allow unlimited header field sizes.

是大于0的正整数。设为0， 表示不限制。

** Warning**

 Setting this parameter to a very high or unlimited value can open up for DDOS attacks.
 该值设置的太大或不限制， 可能会引起DDOS攻击。

Debugging

reload

  • --reload

  • False

Restart workers when code changes.

当代码变化时， 重启工作者进程。

This setting is intended for development. It will cause workers to be restarted whenever application code changes.

该参数是为了开发时使用。只要代码变化， 工作者就会重启。

The reloader is incompatible with application preloading. When using a paste configuration be sure that the server block does not import any application code or the reload will not work as designed.

重载机制和应用的预加载机制不兼容。当使用Paste应用配置时， 请确保服务器没有导入任何的应用代码， 否则重载会出错。

  • spew*

  • --spew

  • False

Install a trace function that spews every line executed by the server.

安装一个跟踪函数， 告知(用户)服务器执行的每一行代码。

This is the nuclear option.

这是终极选项。

check_config

  • --check-config

  • False

Check the configuration.

检查配置是否正确。

Server Mechanics

preload_app

  • --preload

  • False

Load application code before the worker processes are forked.

在工作者进程被fork出之前， 加载应用代码。

By preloading an application you can save some RAM resources as well as speed up server boot times. Although, if you defer application loading to each worker process, you can reload your application code easily by restarting workers.

通过预加载一个应用， 可以节省内存资源、加速服务器启动。然而， 如果将应用的加载延迟到每个工作者进程的创建， 你可以容易的通过重启工作者来重新加载应用。

sendfile

  • --no-sendfile

  • None

Disables the use of sendfile().

禁用sendfile()。

If not set, the value of the SENDFILE environment variable is used to enable or disable its usage.

如果没有设置， 那么环境变量SENDFILE将决定是否启用该参数。

New in version 19.2.

在19.2.中新增的。

Changed in version 19.4: Swapped –sendfile with –no-sendfile to actually allow disabling.

19.4中发生的更改： 使用--no-sendfile替换--sendfile表示禁用。

Changed in version 19.6: added support for the SENDFILE environment variable

19.6中发生的更改： 增加环境变量SENDFILE的支持。

chdir

  • --chdir

  • /home/docs/checkouts/readthedocs.org/user_builds/gunicorn-docs/checkouts/latest/docs/source

Chdir to specified directory before apps loading.

在应用加载前， 将目录切换到指定的路径。

daemon

  • -D, --daemon

  • False

Daemonize the Gunicorn process.

后台运行。

Detaches the server from the controlling terminal and enters the background.

将服务器进程和控制窗口分离，后台运行。

raw_env

  • -e ENV, --env ENV

  • []

Set environment variable (key=value).

设置环境变量。

Pass variables to the execution environment. Ex.:

将变量传给执行环境， 如：

$ gunicorn -b 127.0.0.1:8000 --env FOO=1 test:app

and test for the foo variable environment in your application.

在你的应用中， 测试一下FOO变量。

pidfile

  • -p FILE, --pid FILE

  • None

A filename to use for the PID file.

PID文件的文件名。

If not set, no PID file will be written.

不指定的话， 不会产生PID文件。

worker_tmp_dir

  • --worker-tmp-dir DIR

  • None

A directory to use for the worker heartbeat temporary file.

存放工作者心跳临时文件的目录。

If not set, the default temporary directory will be used.

不指定的话， 会使用默认的临时目录。

user

  • -u USER, --user USER

  • 1005

Switch worker processes to run as this user.

将工作者进程所属的用户切换。

A valid user id (as an integer) or the name of a user that can be retrieved with a call to pwd.getpwnam(value) or None to not change the worker process user.

(需要是)有效的用户id或者是能够通过pwd.getpwnam(value)获取用户的用户名，None表示不切换。

group

  • -g GROUP, --group GROUP

  • 205

Switch worker process to run as this group.

将工作者进程所属的用户组切换。

A valid group id (as an integer) or the name of a user that can be retrieved with a call to pwd.getgrnam(value) or None to not change the worker processes group.

(需要是)有效的用户组id或者是能够通过pwd.getgrnam(value)获取用户组的用户组名称，None表示不切换。

umask

  • -m INT, --umask INT

  • 0

A bit mask for the file mode on files written by Gunicorn.

一个控制Gunicorn产生的文件模式的位掩码。

Note that this affects unix socket permissions.

注意： 这会影响unix套接字的权限。

A valid value for the os.umask(mode) call or a string compatible with int(value, 0) (0 means Python guesses the base, so values like 0, 0xFF, 0022 are valid for decimal, hex, and octal representations)

该值需要对os.umask(mode)有效， 或者兼容int(value, 0)(0表示基)函数。

initgroups

  • --initgroups

  • False

If true, set the worker process’s group access list with all of the groups of which the specified username is a member, plus the specified group id.

该参数如果是true，设置工作进程的组访问列表中的所有组指定的用户名是一个成员，加上指定的组编号。

New in version 19.7.

19.7版本中新增加。

tmp_upload_dir

  • None

Directory to store temporary request data as they are read.

当请求数据被读取时， 临时存储这些数据的目录。

This may disappear in the near future.

不久的将来会出现。

This path should be writable by the process permissions set for Gunicorn workers. If not specified, Gunicorn will choose a system generated temporary directory.

该路径必须是Gunicorn工作者进程可写的。如果没有指定， Gunicorn会选择一个系统生成的临时目录。

secure_scheme_headers

  • {'X-FORWARDED-PROTOCOL': 'ssl', 'X-FORWARDED-PROTO': 'https', 'X-FORWARDED-SSL': 'on'}

A dictionary containing headers and values that the front-end proxy uses to indicate HTTPS requests. These tell Gunicorn to set wsgi.url_scheme to https, so your application can tell that the request is secure.

一个供前端代理用来表明HTTPS请求、存储请求头和对应值的字典。这些参数会告知Gunicorn把wsgi.url_scheme设置成https， 所以你的应用能够识别请求是安全的。

The dictionary should map upper-case header names to exact string values. The value comparisons are case-sensitive, unlike the header names, so make sure they’re exactly what your front-end proxy sends when handling HTTPS requests.

这个地点需要把大写的头域名映射到准确的字符串值。值的比较是大小写无关的， 不像头名称， 所以请确保你的前端代理在处理https请求时，传递了准确的值。

It is important that your front-end proxy configuration ensures that the headers defined here can not be passed directly from the client.

很重要： 你的前端代理不能把这里定义的headers传给客户端

forwarded_allow_ips

  • --forwarded-allow-ips STRING

  • 127.0.0.1

Front-end’s IPs from which allowed to handle set secure headers. (comma separate).

来源于这些IP的前端IP被允许处理设置安全的请求头（逗号分隔）。

Set to * to disable checking of Front-end IPs (useful for setups where you don’t know in advance the IP address of Front-end, but you still trust the environment).

该参数被设置成 *时， 不允许检查前端的IP。

By default, the value of the FORWARDED_ALLOW_IPS environment variable. If it is not defined, the default is “127.0.0.1”.

默认使用FORWARDED_ALLOW_IPS环境变量。 如果该环境变量没有被定义， 默认是 127.0.0.1。

Logging

accesslog

  • --access-logfile FILE

  • None

The Access log file to write to.

记录访问日志的文件路径。

‘-‘ means log to stdout.

‘-‘表示日志输出到标准输出。

access_log_format

  • --access-logformat STRING

  • %(h)s %(l)s %(u)s %(t)s "%(r)s" %(s)s %(b)s "%(f)s" "%(a)s"

The access log format.

访问日志格式。

Identifier  Description
h   remote address
l   '-'
u   user name
t   date of the request
r   status line (e.g. GET / HTTP/1.1)
m   request method
U   URL path without query string
q   query string
H   protocol
s   status
B   response length
b   response length or '-' (CLF format)
f   referer
a   user agent
T   request time in seconds
D   request time in microseconds
L   request time in decimal seconds
p   process ID
{Header}i   request header
{Header}o   response header
{Variable}e environment variable

记录错误日志的文件。

‘-‘表示输出到标准错误输出。

19.2中做出改变： 默认输出到标准错误输出。

输出错误日志的粒度(不低于该级别)。

有效的级别有：

19.6中新增。

logger_class

The logger you want to use to log events in Gunicorn.

你想用来记录事件的日志器。

The default class (gunicorn.glogging.Logger) handle most of normal usages in logging. It provides error and access logging.

默认的类是gunicorn.glogging.Logger， 能够满足大多数的日志需要。它提供错误和访问记录。

You can provide your own worker by giving Gunicorn a Python path to a subclass like gunicorn.glogging.Logger. Alternatively the syntax can also load the Logger class with egg:gunicorn#simple.

你可以通过配置gunicorn.glogging.Logger的子类来提供给工作者新的日志器。 也可以通过egg:gunicorn的语法来加载日志器。

要使用的日志配置文件。 Gunicorn使用标准的Python日志模块配置文件形式。

发送系统消息的地址。

用于UNIX域套接字。 TYPE可以是流驱动接收的stream或者数据报接收的dgram。默认是stream。

用于UDP套接字。

用于TCP套接字。

让Gunicorn在syslog中使用该参数作为程序名。

所有的日志项都会以*gunicorn.*作为前缀。默认情况下， 程序名是进程的名字。

Enable inheritance for stdio file descriptors in daemon mode.

在后台模式下， 启用对标准输入输出文件描述符的继承。

Note: To disable the Python stdout buffering, you can to set the user environment variable PYTHONUNBUFFERED .

注意： 为了禁用Python标准输出的缓冲， 你可以设置PYTHONUNBUFFERED这个环境变量。

host:port of the statsd server to log to.

记录日志的statsd server的host:port。

19.1中新增该参数。

Process Naming

proc_name

  • -n STRING, –name STRING

  • None

A base to use with setproctitle for process naming.

使用该选项需要先安装setproctitle 。

This affects things like ps and top. If you’re going to be running more than one instance of Gunicorn you’ll probably want to set a name to tell them apart. This requires that you install the setproctitle module.

这会影响ps和top命令的显示结果。如果你运行了多个Gunicorn，你很可能想通过名称区分它们。这要求你安装setproctitle 模块。

If not set, the default_proc_name setting will be used.

如果未设置， 会使用*default_proc_name *的设置。

default_proc_name

  • gunicorn Internal setting that is adjusted for each type of application.

为每类应用条结果的内部配置。

Server Mechanics

A comma-separated list of directories to add to the Python path.

逗号分隔的待添加的python路径的目录。

e.g. ‘/home/djangoprojects/myproject,/home/python/mylibrary’.

例如， ‘/home/djangoprojects/myproject,/home/python/mylibrary’

paste

  • –paste STRING, –paster STRING

  • None

Load a PasteDeploy config file. The argument may contain a # symbol followed by the name of an app section from the config file, e.g. production.ini#admin.
