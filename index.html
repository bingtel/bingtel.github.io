Python2.7中 @property 描述符的使用

Python 描述符

描述符定义

Python Descriptor是指： 按照descriptor协议, 有这样的属性之一

def __get__(self, obj, type=None)  #  会返回一个value
def __set__(self, obj, value)   # 返回None
def __delete__(self, obj)  # 返回None

这样的对象就是一个descriptor。

描述符特效

假若有一个对象t, 我们去引用它的一个属性a

t.a

但是发现a是一个descriptor

那么不会返回a, 而是会去调用a相应的get, set, delete

那么什么情况调用那个呢?如下

v = t.a   <---->   v = __get__(a, t)
t.a = v   <----->  __set__(a, t, v)
del t.a   <----->  __delete__(a, t)

描述符实现

只有new-style objects或class的属性在被引用时,descriptor的特性才能起作用

从 object 派生的类就是 new-style class

class T(object):
    pass

@property

Python有内置property()函数, 它可以直接做函数,也可以用来做装饰器, 它的使用方式如下

class Test(object):
    def getx(self):
           return self._x
    def setx(self, v):
           self._x = v
    def deletex(self):
           del self._x

    x = property(getx, setx, deletex, ''' __doc__''')

等价于：

class Test(object):
    @property
    def x(self):
        return self._x

    @x.setter
    def x(self, v):
        self._x = v

    @x.deleter
    def x(self):
        del self._x

使用例子

class Person(object):
    def __init__(self, first_name):
        # 此处使用self.first_name是为了能够使用setter对赋值进行校验
        # 可以换成 self._first_name, 但缺乏校验
        self.first_name = first_name  

    # Getter function
    @property
    def first_name(self):
        return self._first_name

    # Setter function
    @first_name.setter
    def first_name(self, value):
        if not isinstance(value, str):
            raise TypeError('Expected a string')
        self._first_name = value

    # Deleter function (optional)
    @first_name.deleter
    def first_name(self):
        raise AttributeError("Can't delete attribute")

常见错误

网上常见的例子是：

class Person:
    ...

但是会发现，

setter、deleter

并不起作用，是因为，

class Person:

只是个旧式类， 要改成新式类：

class Person(object):
