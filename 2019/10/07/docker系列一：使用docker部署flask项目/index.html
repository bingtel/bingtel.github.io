docker容器

启动容器

我们先测试一下docker镜像的启动，从而来创建一个容器：

docker run -it --rm -p 80:8080 foo:flask_demo

解释：

-i: 和docker交互
-t: 创建一个tty
--rm: 当docker容器退出的时候，就删除容器
-p： 将宿主机的端口映射到docker容器的端口

查看容器

新开一个命令行窗口，通过:

docker ps -a

可以看到所有的容器列表，

注意： 

容器的id是dae114ea6d7f。
我们启动容器时，并未给容器指定名称，docker会随机生成一个。
docker ps 只显示处于运行状态的容器。

查看容器日志

通过：

docker logs dae114ea6d7f

即可查看该容器的日志。

登录容器

使用：

docker exec -it dae114ea6d7f /bin/bash

即可登录进入容器。

注意：

关闭容器:

可以通过ctrl + c或者docker stop dae114ea6d7f停止容器的运行。

生产环境部署docker

需要-d：

docker run -d -p 80:8080 foo:flask_demo

来后台运行docker容器。

删除容器

docker rm <容器id>

删除镜像

docker rmi <镜像id>

问题来了：

是容器没有启动成功还是启动后就退出来呢？

答案是后者，但为什么容器启动后就退出呢？

因为：

因为Docker容器仅在它的1号进程（PID为1,在这个例子中，就是gunicorn的master进程）运行时，会保持运行。如果1号进程退出了，Docker容器也就退出了.

所以，要使容器不退出必须有一个前台执行的进程.gunicorn变为daemon模式后，会使得容器启动后立即退出。还是要：

# daemon = True
