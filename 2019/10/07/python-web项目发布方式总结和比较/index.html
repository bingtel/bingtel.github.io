B. 压缩包方式

该方式相比于上种方式，将主要工作在线下完成：

  1. 线下完成主要工作，减少了线上发布的时间
  2. 发布失败，可以回滚到上一版本的稳定服务。 无法将项目作为别的项目的依赖发布 将web app打成安装包，上传到pypi服务器中，这样别的项目可以安装该包，实现复用。
  3. 把web项目打成安装包，并标记新的版本号
  4. 搭建私有的PYPI服务器，用于部门内部存放私有的安装包
  5. 线上环境，安装新的版本的项目安装包；如果发布失败，安装前一版本的包即可。 virtualenv venv
    source venv/bin/activate
    pip install pypiserver mkdir pkgs
    pypi-server -p 8080 pkgs/
  6. pkgs目录用于存储日后内部发布的私有包
  7. pypiserver并不会换成PYPI镜像 Welcome to pypiserver!
